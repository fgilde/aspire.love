How it works

Three steps from cloud-bound to fully local.

1

Point it at your project

Run the CLI or open the desktop app and select your Lovable project folder. The project name is read from package.json.

2

Pick a mode & options

Choose Full Local, Supabase Sync or Remote Connect, set your default user, and optionally enable observability.

3

Generate & run

An aspire folder with a Solution and AppHost is generated. Hit aspire run and everything comes up together.

Three modes, one tool

Start fully local, switch later — the generated code is commented to make switching trivial.

The engine under the hood

aspire.love doesn't reinvent Supabase orchestration — it builds on the open-source Nextended.Aspire.Hosting.Supabase integration, which makes the whole Supabase stack a first-class .NET Aspire citizen.

What the package does

One builder.AddSupabase(...) call spins up Postgres, Auth, Storage, Realtime, the Kong gateway, Studio and the edge-function runtime as managed Aspire resources — with a fluent API for migrations, edge functions, a seeded user, cloud sync and observability.

Why aspire.love uses it

It's the engine behind every mode. aspire.love generates a clean, readable AppHost around it instead of hand-rolled Docker Compose — so you get a real .NET solution you can read, tweak and extend with the package's own API.

View the package on NuGet 
// Postgres · Auth · Storage · Realtime · Studio · edge functions
var supabase = builder.AddSupabase("supabase")
    .ConfigureStudio(s => s.WithProjectName("MyApp"))
    .WithRegisteredUser(email, password, name)
    .WithMigrations(migrationsPath)
    .WithEdgeFunctions(edgeFunctionsPath);

Frequently asked questions

Everything you might wonder before running it on your project.

Does it change my Lovable project?

Almost nothing is touched. aspire.love adds a self-contained aspire/ folder and a single aspire script to your package.json so the AppHost can launch the frontend. Your existing source stays exactly as it is.

What do I need to run it?

The .NET 10 SDK, Docker (for the local Supabase stack) and the .NET Aspire workload. The generated AppHost orchestrates everything from there.

Can I switch modes later?

Yes. Start with Full Local and move to Supabase Sync or Remote Connect whenever you like — the generated C# is commented to make switching by hand trivial, or just re-run the tool.

Is it free and open source?

Completely. aspire.love is MIT-licensed and developed in the open on GitHub. Use it for personal and commercial projects alike.

Windows only?

The aspire.love Studio desktop app is Windows-only, but the love.aspire CLI runs anywhere .NET 10 runs — Windows, macOS and Linux.

Does Lovable's built-in AI still work?

Yes. Provide your LOVABLE_API_KEY and the local edge runtime keeps your project's AI features working while everything runs on your machine.
